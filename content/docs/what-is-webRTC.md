    WebRTC 是什么？ 📡

    WebRTC（Web Real-Time Communication）是一种让浏览器之间直接进行实时音视频通话和数据传输的技术标准，由 Google 主导并已被 W3C 纳入国际标准。

    核心特点

    - P2P 直连 — 两个浏览器不通过服务器直接通信（也叫端到端连接），延迟更低
    - 内置浏览器 — 无需安装任何插件或软件，直接在网页里就能用
    - 音视频通话 — 支持语音通话、视频通话、屏幕共享等场景
    - 数据传输 — 除了音视频，还能传输任意二进制数据（比如文件、文本）

    典型应用场景

    | 场景 | 举例 |
    |------|------|
    | 视频通话 | Google Meet、钉钉视频会议 |
    | 直播连麦 | 抖音/B站直播中的连麦互动 |
    | 远程控制 | 浏览器里的远程桌面 |
    | 文件传输 | 浏览器间直接传文件（不需要上传到服务器） |
    | 游戏 | 实时多人游戏的数据同步 |

    简化的工作原理


    浏览器 A <──── NAT/防火墙 ─---> 浏览器 B
      │                                  │
      └────── ICE 协议协商路径 ----------┘
               ↓
        直接建立 P2P 连接（或通过 TURN 中继）
               ↓
        音视频/数据直接传输


    主要涉及的技术组件：
    - getUserMedia — 获取摄像头/麦克风
    - RTCPeerConnection — 建立 P2P 连接的核心 API
    - RTCDataChannel — 传输任意数据

    一个简单例子 🧪

    javascript
    // 获取本地媒体（摄像头+麦克风）
    navigator.mediaDevices.getUserMedia({ video: true, audio: true })
      .then(stream => {
        // 把本地视频流显示到自己的页面上
        myVideo.srcObject = stream;
      });


    与 WebSocket 的区别

    | | WebRTC | WebSocket |
    |--|--------|-----------|
    | 连接方式 | P2P 直连 | 需要服务器中转 |
    | 延迟 | 极低（实时） | 相对较高 |
    | 适用场景 | 音视频通话、实时数据 | 消息推送、状态同步 |
    | 连接性质 | 可持久，也可一次性的 |

    总的来说，WebRTC 是浏览器原生支持的"打电话"技术，让网页拥有了接近原生应用的实时通信能力，是现代视频通话、直播、在线协作等应用的重要基石 💪
